Universal Registration Form

  

Universal Registration Form

  

One Registration for can be configured for the entire application. The idea is that this form is the universal registration for the facility. Different departments may have their own enrollment form which you would recreate as a dynamic form builder form.

 

The pre-defined registration fields are saved in two different tables - mst_patient and dtl patientcontact. Any new custom fields you create will be placed in a new table called DTL_FBCUSTOMFIELD_Patient_Registration.

 

There is a minimum set of fields that must be captured in order to register the patient into the system.
